The admin panel is a local-only editor that runs alongside your dev server. It reads and writes the same `.mdx` files you'd edit by hand — no database, no sync, just files. It's designed for comfortable writing without switching to your code editor for every metadata change.

<Callout type="warn" title="Dev only">
  The admin panel is only available at `http://localhost:4321/admin/` during local development. It's excluded from production builds via the Astro sitemap config.
</Callout>

## Opening the panel

Start the dev server and navigate to the admin URL:

```bash
bun run dev
# → open http://localhost:4321/admin/
```

## Layout overview

The panel has three columns:

1. **Left sidebar** — list of all posts with search, date, and tags. Click a post to load it.
2. **Main panel** — the editor. Switch between **Content** (MDX editor) and **Frontmatter** (metadata form) tabs.
3. **Right panel** — switch between **Preview** (live render), **Components** (insert snippets), and **Images** (upload and manage).

## The content editor

The editor is CodeMirror with Markdown syntax highlighting. Write standard Markdown and MDX — component imports, JSX, everything.

Keyboard shortcuts:
- <Kbd>⌘</Kbd> + <Kbd>S</Kbd> — force save immediately
- Autosave triggers 900ms after you stop typing

<Callout type="info" title="Inserting components">
  Open the **Components** tab in the right panel, find a component, and click **Insert**. The snippet is inserted at your cursor position and the editor briefly highlights where it landed.
</Callout>

## The frontmatter tab

Switch to **Frontmatter** to edit metadata without touching YAML. The form includes:

- **Slug** — editable, with a lock toggle. Unlocked: auto-follows the title. Locked: edit independently.
- **Title** and **Description** — with character meters showing SEO status.
- **Search preview** — live rendering of how the post appears in Google.
- **Hero image** — text input plus a picker that shows images already uploaded for this post.
- **Publish date** and **Updated date** — date pickers.
- **Author** and **Tags** — tag badges (press Enter or comma to add).
- **Draft** and **Featured** toggles.

## Live preview

The **Preview** tab shows a full render of the post as it will appear on the site — including custom components, syntax highlighting, and styles. It updates on every autosave.

## Deleting a post

Click **Delete** in the top-right of the main panel. A confirmation dialog appears — the action permanently removes the `.mdx` file. There's no undo (use git to recover).

## Sidebar

The sidebar lists all posts with:
- **Title** and **draft/featured badges**
- **Publish date**
- **Tags** (up to 3 shown, `+N` for the rest)

Use the search box to filter by title, slug, or tag.

<InfoCard title="Files are just files" icon="📁">
  Everything the admin panel does is a read or write to the `src/data/blog/` directory. You can edit files in your code editor at the same time — the panel reloads on file changes.
</InfoCard>
